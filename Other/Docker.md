Контейнер - это облегчённая, изолированная среда, которая упаковывает ваше приложение вместе со всем необходимым для его работы: кодом, средой выполнения, библиотеками и конфигурацией.
Контейнеры отличаются от виртуальных машин. Виртуальная машина виртуализирует весь аппаратный стек и нуждается в собственном ядре операционной системы. Контейнеры используют ядро ​​хостовой ОС и гораздо легче - они запускаются за миллисекунды и используют лишь малую часть памяти.

## Работа с контейнером
Команды:
- `docker run -d -p 8080:80 --name my-nginx nginx`  (запуск контейнера)

| Часть команды     | Значение                                                                    |
| ----------------- | --------------------------------------------------------------------------- |
| `-d`              | Detached - работает в фоновом режиме, возвращая терминал пользователю       |
| `-p 8080:80`      | Переадресация портов - переадресация порта хоста 8080 на порт контейнера 80 |
| `--name my-nginx` | Название контейнера                                                         |
| `nginx`           | Название образа                                                             |
- `docker ps` (список запущенных контейнеров)
- `docker ps -a` (список всех контейнеров)
- `docker logs my-nginx` (логи контейнера)
- `docker exec my-nginx ls /usr/share/nginx/html` (запуск команд внутри контейнера)
- `docker exec -it my-nginx bash` (открытие интерактивной оболочки внутри контейнера)
- `docker stop my-app` (остановка контейнера)
- `docker rm my-app` (удаление контейнера)
## Образ VS контейнер

|              | Образ                          | Контейнер                                   |
| ------------ | ------------------------------ | ------------------------------------------- |
| Что это      | Неизменяемый шаблон            | Запущенный или остановленный инстанс образа |
| Создание     | `dockeer build`/`dockeer pull` | `docker run`                                |
| Перечисление | `docker images`                | `docker ps -a`                              |
| Удаление     | `docker rmi`                   | `docker rm`                                 |
Из одного и того же образа можно создать несколько контейнеров — каждый из них получает свой собственный изолированный записываемый слой поверх общих слоев образа, доступных только для чтения.
## Dockefile
Dockerfile - это рецепт для создания образа контейнера. Docker считывает каждую инструкцию сверху вниз, выполняет каждую из них и фиксирует результат как новый слой. Слои накладываются друг на друга, образуя окончательный образ.

| Инструкция | Что делает                                                       |
| ---------- | ---------------------------------------------------------------- |
| FROM       | Существующий базовый образ                                       |
| WORKDIR    | Установка рабочего каталога внутри образа                        |
| COPY       | Копирование файлов из контекста сборки в образ                   |
| RUN        | Выполнение команды оболочки во время сборки                      |
| EXPOSE     | Порт, на котором приложение будет прослушивать запросы           |
| CMD        | Команда по умолчанию, которая запускается при запуске контейнера |
| USER       | Пользователь, под которым будет выполняться контейнер            |
- `docker build -t <image_name> .` (создание образа)
Образы неизменяемы. Для обновления приложения всегда создается новый образ, после чего контейнер заменяется новым. Это делает развертывание предсказуемым, а откат - простым.
Образы Docker — это не отдельные файлы, а наборы слоев, доступных только для чтения. Каждая инструкция Dockerfile, изменяющая файловую систему, создает новый слой поверх предыдущих. При запуске контейнера Docker добавляет один тонкий слой, доступный для записи, поверх всех изменений, внесенных контейнером.  
Такая многоуровневая структура делает Docker быстрым: если слой не изменился с момента последней сборки, Docker повторно использует кэшированную версию вместо пересборки. Поэтому в начало Dockerfile добавляют слои, которые будут редко изменяться.
- `docker history <image_name>` (получение слоев образа)
## Docker Compose
Docker Compose - это инструмент для определения и запуска многоконтейнерных приложений. Вместо того чтобы управлять каждым контейнером вручную, можно описать весь стек вашего приложения в одном файле `compose.yaml` и запустить все одной командой.  
Docker Compose использует конфигурационный файл YAML (обычно `compose.yaml` /`docker-compose.yml`) для определения многоконтейнерных приложений. 
Стандартный файл Compose организован в несколько разделов верхнего уровня:
- `services`: Определяет отдельные контейнеры, из которых состоит ваше приложение.
- `networks`: Настраивает общие сети для связи между сервисами.
- `volumes`: Определяет постоянное хранилище, совместно используемое хостом и контейнерами или между сервисами.
- `configs`/`secrets`: Управляет файлами конфигурации или конфиденциальными данными (например, паролями), монтируемыми в контейнеры.
Каждый сервис может включать следующие общие директивы:
- `image`: Указывает предварительно собранный образ Docker для использования (например, `image: redis:alpine`).
- `build`: Указывает Compose собрать образ из локального Dockerfile.
- `ports`: Сопоставляет порты хоста с портами контейнера (например, `8080:80`).
- `environment`: Устанавливает переменные среды внутри контейнера.
- `depends_on`: Указывает зависимости порядка запуска между службами.
- `restart`: Определяет политику перезапуска (например, всегда, если не остановлен).
Расширенные синтаксические возможности:
- Интерполяция переменных: `${VARIABLE}` используется для внедрения значений из файла `.env` или среды оболочки в YAML.
- Включить/Расширить: `include` применяется для модульной организации больших проектов путем ссылки на другие файлы Compose или `extends` для повторного использования конфигураций служб.
- Режим отслеживания: В разделе `develop` можно определить блок отслеживания для автоматической синхронизации изменений локальных файлов с работающим контейнером для разработки в реальном времени.
## Работа с Docker Compose
- `docker compose up --build -d` (создание контейнера из образа и его запуск)
- `docker compose up -d` (запуск контейнера)
- `docker compose ps` (список запущенных контейнеров из файла Docker Compose)
- `docker compose logs <container_name>` (логи контейнера)
- `docker compose down -v` (остановить контейнер и удалить тома (volumes))
- `docker compose config` (полная конфигурация)
